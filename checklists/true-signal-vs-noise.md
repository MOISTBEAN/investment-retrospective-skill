# True Signal vs Noise Checklist

Use this checklist to avoid hindsight bias and narrative overfitting. Evaluate each item using only the information available on its date before considering later confirmation.

## Mandatory signal gate

Classify an event as a **true signal** only when all three gates pass:

1. **Relevance:** It directly changes or tests the assumption behind the documented market fear.
2. **Evidence:** A Grade A source, or a Grade B source plus independent confirmation, supports the claim.
3. **Persistence:** A later event, reported result, or durable benchmark-adjusted market reaction reinforces the interpretation.

Price confirmation is useful but not mandatory. If relevance passes but evidence or persistence does not, use **possible signal**. If relevance fails, use **noise**. If the record is incomplete, use **too uncertain**.

## True signal characteristics

A signal is stronger when:

- [ ] It directly addresses the original market fear.
- [ ] It changes a key forward-looking assumption.
- [ ] It is supported by a high-quality source.
- [ ] It is confirmed by independent evidence.
- [ ] The stock shows abnormal return versus a suitable peer set or index.
- [ ] Follow-up events reinforce the same narrative.
- [ ] Analysts or market participants begin changing the valuation framework.
- [ ] The event affects revenue, margin, customer adoption, competitive position, financing, or business-model durability.
- [ ] The information was specific and knowable at the time.
- [ ] The interpretation survives plausible alternative explanations.

## Noise characteristics

A signal is weaker when:

- [ ] It is only loosely related to the core thesis.
- [ ] It relies on generic headlines, broad AI/news-cycle excitement, or undifferentiated sector narratives without a specific event window, benchmark-relative move, and independent evidence layer.
- [ ] It does not change forward expectations.
- [ ] It is management optimism without external validation.
- [ ] It was already widely expected.
- [ ] The stock move is mostly sector beta.
- [ ] It is a one-day reaction with no follow-through.
- [ ] It comes from low-quality or speculative sources.
- [ ] It sounds important only with hindsight.
- [ ] Its apparent importance depends on facts published later.
- [ ] Equally plausible alternatives explain the move.

## Questions to ask

- Did this event change what the market could reasonably believe at the time?
- Did it resolve or confirm a prior fear?
- Did it create a genuinely new narrative or merely rename an existing one?
- Did price action show stock-specific interest after benchmark adjustment?
- Was the reaction durable across the relevant window?
- Could macro, sector, short covering, positioning, index flows, or multiple expansion explain the move instead?
- Was the information knowable at the time?
- What confirming evidence would have been required before acting on the interpretation?
- What observable result would falsify the interpretation?
- Would the same reasoning classify a similar event as important if the stock had later fallen?

## Classification guide

- **True signal:** Material, contemporaneously knowable, well sourced, and reinforced by independent or later evidence.
- **Possible signal:** Relevant and knowable, but missing confirmation or facing strong alternatives.
- **Noise:** Weakly related, non-material, expected, low quality, or largely explained by broader factors.
- **Too uncertain:** Evidence is insufficient to classify responsibly.

Do not label an early signal “noise” merely because the thesis failed. Judge process quality separately from outcome.
